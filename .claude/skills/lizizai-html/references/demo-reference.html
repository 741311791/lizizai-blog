这是一篇完整的参考示例，展示 lizizai-html Skill 生成的 HTML 文件的所有功能。当你使用这个 Skill 时，生成的文件将嵌入到博客文章页的 iframe 中，右侧边栏会自动显示 HTML 内容的目录结构。

设计系统一致性

每份 HTML 文件必须引用博客的主题 CSS 和字体，确保视觉风格与主站完全一致。主题 CSS 提供了所有基础排版样式——标题、段落、代码块、引用等——无需手动编写。

支持的容器宽度有三个：

  • .prose（默认）— 680px 居中，适合文字阅读
  • .prose-wide — 960px，适合数据图表
  • .prose-full — 100% 宽度，适合全宽展示

颜色体系

所有颜色使用 CSS 变量定义。自定义样式时必须使用变量而非硬编码值：

正确：var(--color-primary)　错误：#d97706
正确：var(--color-surface)　错误：#18181b

功能边界

使用此 Skill 时，需要明确它做什么和不做什么：

✅ Skill 职责范围

  • 生成与博客设计系统视觉一致的独立 HTML 文件
  • 确保主题 CSS、字体、postMessage 脚本正确嵌入
  • 自动提取标题结构，通过 postMessage 发送给父页面
  • 提供组件类（card、callout、badge 等）供内容编排
  • 文件自包含，可直接上传 R2

❌ 不属于 Skill 职责

  • 父页面布局 — 侧边栏渲染、TOC 高亮、滚动追踪由前端组件处理
  • 文件上传部署 — 上传到 R2 是部署流程，非 Skill 生成逻辑
  • 响应式适配 — iframe 宽度由父页面控制，HTML 内容自身不处理
  • SEO / 元数据 — 由 Next.js 页面层处理，iframe 内容无 SEO 价值
  • 用户交互 — 评论、点赞、分享等由父页面组件处理
  • 内容路由 — 文章数据管理、URL 路由由 Next.js App Router 处理

🔗 协作边界

Skill 生成的 HTML 与父页面通过 postMessage 协议通信：

消息 方向 用途
html-content-height iframe → 父 动态调整 iframe 高度
html-toc iframe → 父 标题结构 + 位置信息

父页面负责：渲染侧边栏目录、计算活跃标题、处理点击跳转。Skill 只管生成正确的 HTML 和发送正确的消息。

关于标题 ID — Skill 的 TOC 脚本会自动为没有 id 的标题生成 id。如果你手动指定了 id，脚本会保留。id 生成规则：heading-{text-slug}，重复时追加 -2、-3 等后缀。

编写最佳实践

结构建议

一篇好的 HTML 内容应该：

  1. 使用 h2 作为章节标题，h3 作为子节——这决定了侧边栏的目录层级
  2. 避免使用 h1——文章标题由父页面渲染
  3. 章节之间保持足够的间距（主题 CSS 自动处理）
  4. 合理使用 callout 突出重点，不要过度使用

性能注意

  • 不要引入外部 JS 库（jQuery、Chart.js 等）—— 保持零依赖
  • 图片使用标准 <img> 标签，高度同步脚本会自动处理
  • 如果内容很长（超过 2 倍视口高度），iframe 高度会被父页面限制
