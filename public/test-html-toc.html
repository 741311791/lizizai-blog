第一章：HTML TOC 测试

这是第一章的内容。用于测试 HTML 内容的 TOC 目录提取功能。

当用户切换到 HTML 内容类型时，侧边栏应该显示 HTML 内容自身的目录结构，而不是 Markdown 文章的目录。

1.1 标题提取验证

这一节验证 h3 级别的标题是否被正确提取。

每个标题都应该有一个自动生成的 ID，用于目录跳转。

1.2 位置计算验证

这一节验证标题的 top 位置是否正确计算。

getOffsetTop 函数需要遍历 offsetParent 链来获取绝对偏移。

第二章：滚动追踪测试

这是第二章的内容。父页面通过 scroll 事件和标题位置来计算当前可见的标题。

当用户滚动页面时，侧边栏应该高亮对应的目录项。

2.1 视口计算

验证视口位置计算逻辑：viewportTopInIframe = -iframeRect.top + headerOffset

2.2 活跃标题匹配

验证找到最后一个 top <= viewportTopInIframe 的标题作为活跃标题。

第三章：点击跳转测试

这是第三章的内容。点击侧边栏目录项时，应该滚动到对应标题位置。

跳转目标：iframe.top + heading.top - headerOffset

3.1 平滑滚动

验证 scrollTo 使用 smooth 行为。
